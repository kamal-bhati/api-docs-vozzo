## Complete Authentication Flow

### Step 1: Access API Documentation
When you attempt to access the API reference documentation at `/api`, the system automatically detects authentication requirements and redirects you to the secure login portal.

### Step 2: OpenID Connect Authentication
You are redirected to Vozzo's authentication server where you enter your credentials. The system validates your identity and establishes a secure session using OpenID Connect protocol.

**Authentication Configuration:**
- **Issuer**: https://auth.vozzo.ai/realms/vozzo-dev
- **Client ID**: cc57a815-f859-fefc-66a7-d97280cd799e
- **Scopes**: openid, profile, email
- **Protected Routes**: /api/*, /v1alpha/*

### Step 3: API Documentation Access
After successful authentication, you are redirected back to the API documentation with full access to all protected endpoints and examples.

### Step 4: Generate API Keys
From the authenticated session, you can generate API keys for programmatic access:

```bash
curl -X POST "https://backend.dev.vozzo.ai/v1alpha/auth/api-keys" \
  -H "Authorization: Bearer {your_access_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "scopes": ["campaign:read", "campaign:write"],
    "expiry_date": "2024-12-31",
    "name": "My API Key"
  }'
